The virtual viewport

  • 320x480 virtual screen.

    These are CSS pixels.

  • Set font size for <html>.

    html { font: 62.5% "MozTT"; }

  • Default UA font size is 16px.

    62.5% * 16px = 10px

  • Now use rem units.

    rem stands for "root em"

  • Proportional to html font size.

    1rem = 10px; 2rem = 20px;

Example of CSS setup


* {
  padding: 0;
  margin: 0;
}

html {
  font-size: 62.5%;
  font-family: "MozTT", Sans;
  overflow: hidden;
}

body, p, td, th, pre, body{
  font-size: 1.5rem;
}

Multi-views applications

  • Apps have several views.

    These views are like "cards".

  • Listen for hash changes.

    Each hash is a view configuration.

  • "data-viewport" attribute.

    To control position.

  • CSS attribute selectors.

    [data-viewport="right"] { }

  • CSS transform rules.

    { transform: translateX(100%) }.

Markup example

<section id="test" role="region" class="card" data-viewport="right">
  <header>
    <h1>Title example</h1>
  </header>
</section>

CSS example

.card {
  position: absolute;
  top: 0; bottom: 0; left: 0; right: 0;
  transition: transform 0.3s ease;
  transform: translate(0,0);
}
.card[data-viewport="right"] {
  transform: translateX(100%);
}

JavaScript example

window.addEventListener('hashchange', function () {
  // dispose the current view
  currentView.dataset.viewport = currentView.dataset.origin;

  // bring the target view
  var target = location.hash.split('/')[1];
  currentView = document.getElementById(target);
  currentView.dataset.origin = target.dataset.viewport;
  delete currentView.dataset.viewport;
});
